# 任务 P021：重组 ProMax.yaml rules 段分组顺序

## 描述

对 `mihomo/config/ProMax.yaml` 中 `rules` 段的分组顺序和注释进行优化调整：
- 提取 `Cusproxy` 和 `Public-tracker` 归入独立的自定义分组
- 将 `Cloudflare` 移至 Apple/Microsoft 附近的通用服务分组
- 交换国内流量与国外流量的分组顺序，使国外流量规则优先于国内流量
- 优化注释分组标签

## 背景

原有 rules 段分组顺序存在改进空间：
- `Cusproxy`（自定义代理）混在 AI/Telegram 分组中，逻辑不够清晰
- `Public-tracker`（公共 BT  tracker）在通用国外流量分组中，归类不准确
- `Cloudflare` 位于 AI/Telegram 分组后，与 Apple/Microsoft 同类服务分居两处
- 国内流量分组排在国外流量之前，但实际匹配逐条从上到下执行，将国外流量提前可减少非必要延迟

## 修改内容

### 文件列表

| 文件 | 变更类型 |
|---|---|
| `mihomo/config/ProMax.yaml` | 移项、分组重排、注释优化 |

### 具体变更

| # | 操作 | 涉及规则 | 说明 |
|---|---|---|---|
| 1 | 移出 | `Cusproxy` | 从原第 3 节（AI/Telegram 组）移至新增的"自定义"节 |
| 2 | 移入 | `Public-tracker` | 从原第 8 节（国外流量组）移至"自定义"节 |
| 3 | 移动 | `Cloudflare` | 从原第 3 节移至 Apple/Microsoft 分组后 |
| 4 | 交换 | 国内流量 ↔ 国外流量 | 国外流量规则优先于国内流量执行，减少非必要延迟 |
| 5 | 优化 | 分组注释 | 更新节标签和编号，补充"自定义"分组说明 |

### 修改前分组顺序

```
# 3. 国外工具与服务
  (Ai, AiIP, Telegram, TelegramIP, Cloudflare, Cusproxy)
# 4. 专属流媒体与社交
# 5. 通用流媒体
# 6. 通用服务
  (Apple, AppleIP, Microsoft)
# 7. 通用国内流量
  (China, ChinaIP)
# 8. 通用国外流量
  (Public-tracker, Proxy, ProxyIP)
# 9. 兜底规则
```

### 修改后分组顺序

```
# 3. 国外工具与服务
  (Ai, AiIP, Telegram, TelegramIP)
# 4. 专属流媒体与社交
# 5. 通用流媒体
# 6. 通用服务
  (Apple, AppleIP, Microsoft, Cloudflare)
# 7. 自定义
  (Public-tracker, Cusproxy)
# 8. 通用国外流量
  (Proxy, ProxyIP)
# 9. 通用国内流量
  (China, ChinaIP)
# 10. 兜底规则
```

## 注意事项

1. 仅调整分组顺序和注释，**不修改**任何规则集内容或策略组归属
2. 保持 YAML 语法正确，空行分隔逻辑分组
3. 规则匹配语义**不变**——仅改变执行顺序，每条规则集的目标策略组（`Proxy` / `China` / `REJECT` 等）不变
