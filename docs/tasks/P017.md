# 任务 P017：新增 PayPal 策略组及同步 hutaorules 仓库新规则

## 描述

扫描 hutaorules 仓库 `rules` 分支中的 `.mrs` 规则文件，与当前配置文件对比，提取新增规则并添加到 `rule-providers` 中。同时新建 PayPal 策略组（us-first 路由）。

## 分析结果

### hutaorules 仓库现有规则（相对于 ProMax.yaml 的新增项）

| 新增规则 | 类型 | 说明 | 路由策略建议 |
|---|---|---|---|
| Paypal | domain | PayPal 支付 | 新建 Paypal 组，us-first |
| Ai-cn | domain | 中国大陆 AI 服务 | China（直连） |
| Cloudflare | domain | Cloudflare 服务 | 新建 Cloudflare 组，us-first |
| Googlefcm | domain | Google Firebase Cloud Messaging | 复用 Google 组 |
| Public-tracker | domain | BT 公共 Tracker | 新建 Public-tracker 组，oversea |

所有 ipcidr 类规则已在 ProMax.yaml 中，无新增。

## 修改内容

### 文件 1: `mihomo/config/ProMax.yaml`

#### 1a. proxy-groups 新增策略组

在 `# 5. 特殊垂直领域` 区域（Crypto 之后，Develop 之前）添加 Paypal：

```yaml
  - { name: Paypal, type: select, proxies: *us-first, icon: https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/Paypal.png }
```

在 `# 5. 特殊垂直领域` 区域（Develop 之后，Speedtest 之前）添加 Ai-cn：

```yaml
  - { name: Ai-cn, type: select, proxies: *direct, icon: https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/AI.png }
```

在 `# 6. 大厂特定服务` 区域（Apple 之后，注释 `# 8.` 之前）添加 Cloudflare：

```yaml
  - { name: Cloudflare, type: select, proxies: *us-first, icon: https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/Cloudflare.png }
```

在 `# 8. 通用国外流量` 区域（Proxy 之后，注释 `# 9.` 之前）添加 Public-tracker：

```yaml
  - { name: Public-tracker, type: select, proxies: *oversea, icon: https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/Torrent.png }
```

#### 1b. rules 新增规则

在 `# 5. 特殊垂直领域` 区域（`RULE-SET,Crypto,Crypto` 之后）添加：

```yaml
  - RULE-SET,Paypal,Paypal
```

在 `# 5. 特殊垂直领域` 区域（`RULE-SET,Dev,Develop` 之后）添加：

```yaml
  - RULE-SET,Ai-cn,China
```

在 `# 6. 大厂特定服务` 区域（`RULE-SET,Microsoft,Microsoft` 之后）添加：

```yaml
  - RULE-SET,Cloudflare,Cloudflare
  - RULE-SET,Googlefcm,Google
```

在 `# 8. 通用国外流量` 区域（`RULE-SET,Proxy,Proxy` 之后）添加：

```yaml
  - RULE-SET,Public-tracker,Public-tracker
```

#### 1c. rule-providers 新增规则集

在 rule-providers 区域末尾（`YoutubeIP` 之后）添加：

```yaml
  Paypal:         { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Paypal.mrs, proxy: China }
  Ai-cn:          { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Ai-cn.mrs, proxy: China }
  Cloudflare:     { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Cloudflare.mrs, proxy: China }
  Googlefcm:      { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Googlefcm.mrs, proxy: China }
  Public-tracker: { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Public-tracker.mrs, proxy: China }
```

#### 1d. fake-ip-filter 新增直连 rule-set 引用

Ai-cn 指向 China（直连策略），应在 fake-ip-filter 中添加 `'rule-set:Ai-cn'`。

ProMax.yaml 原值：
```yaml
fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com, 'rule-set:Private', 'rule-set:Applecn', 'rule-set:Embydirect', 'rule-set:China']
```

目标值：
```yaml
fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com, 'rule-set:Private', 'rule-set:Applecn', 'rule-set:Embydirect', 'rule-set:China', 'rule-set:Ai-cn']
```

## 变更总结

| 文件 | 变更项 |
|---|---|
| ProMax.yaml | 新增 Paypal、Ai-cn、Cloudflare 策略组；新增 5 条规则路由；新增 5 个 rule-providers；fake-ip-filter 增加 Ai-cn |
| Pro.yaml | 新增 Paypal 策略组、规则路由和 rule-provider |
| Standard.yaml | 无变更 |

## 注意事项

1. 新增的 rule-providers URL 需使用 hutaorules 仓库当前路径格式：`https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/XXXX.mrs`
2. 直连规则（Ai-cn -> China）需加入 fake-ip-filter，避免被 fake-ip 拦截
3. 所有新增规则集 `proxy: China` 通过国内代理拉取，与现有规则集一致
4. Paypal 图标使用同名 `Paypal.png`（需确认 Qure 图标集是否存在，若不存在可改用 `Global.png` 或 `Cryptocurrency_3.png`）
5. Public-tracker 图标使用 `Torrent.png`
6. Cloudflare 图标使用 `Cloudflare.png`
7. Ai-cn 复用 AI 图标 `AI.png`
