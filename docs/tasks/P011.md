# P011: 配置中国大陆直连优化（健康检查、测速、规则集源）

## 任务目标

替换三个配置文件中在中国大陆无法直连或连通性差的 URL，确保首次启动时能正常拉取订阅、规则和资源。

## 修改范围

| 文件 | 操作 |
|---|---|
| `mihomo/config/ProMax.yaml` | 替换锚点 URL + 规则集 URL |
| `mihomo/config/Pro.yaml` | 替换锚点 URL + 规则集 URL |
| `mihomo/config/Standard.yaml` | 替换锚点 URL + 规则集 URL |
| `docs/tasks/check_urls.py` | 删除（临时分析脚本，已不需要） |

## 变更内容

### 变更 1：健康检查地址（影响所有 proxy-provider）

**位置**：三个文件中 `x-base-provider` 锚点的 `health-check.url`

| 文件 | 当前值 | 目标值 |
|---|---|---|
| 全部 | `https://www.google.com/generate_204` | `https://cp.cloudflare.com/generate_204` |

Cloudflare 在国内有 CDN 节点，连通性好。同时华为 `connectivitycheck.platform.hicloud.com` 也是一个备选。

### 变更 2：测速/故障转移地址

**位置**：三个文件中 `x-url-test`、`x-fallback`、`x-load-balance` 锚点的 `url`

| 当前值 | 目标值 |
|---|---|
| `https://www.gstatic.com/generate_204` | `https://cp.cloudflare.com/generate_204` |

在三个文件中各出现 2~3 次（具体出现次数与实际文件锚点数量一致）。

### 变更 3：规则集源（rule-providers URL）

**位置**：三个文件中所有 `rule-providers` 的 `url` 字段

**替换规则**：

```
raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/...
↓
cdn.jsdelivr.net/gh/huyanluanyuqaq/hutaorules@rules/...
```

jsDelivr 在国内有节点，拉取速度快很多。

**注意**：逐条手动替换，确保路径转换正确。`@rules` 替换 `refs/heads/rules` 部分，后面的 `mihomo/domain/xxx.mrs` 或 `mihomo/ip/xxx.mrs` 或 `mihomo/ipcidr/xxx.mrs` 保持不变。

### 变更 4：删除临时分析脚本

删除 `docs/tasks/check_urls.py`（已无用途）。

## 禁止修改项

- 不修改 `ui/` 目录或 zashboard URL（UI 下载一次后本地缓存）
- 不修改图标 URL（GitHub icons 不影响功能）
- 不修改 `docs/`、`PROJECT_LESSONS.md`、`PROJECT-SPEC.md`、`AGENTS.md`、`CLAUDE.md`
- 不修改 `mihomo/openclash/` 下的 `.conf` 文件
- 不修改规则集内容、策略组定义、DNS 配置

## 验收标准

1. 三个配置文件中不再出现 `google.com/generate_204`（全部替换为 `cp.cloudflare.com`）
2. `gstatic.com/generate_204` 全部替换为 `cp.cloudflare.com/generate_204`
3. `raw.githubusercontent.com/huyanluanyuqaq/hutaorules/...` 全部替换为 `cdn.jsdelivr.net/gh/huyanluanyuqaq/hutaorules@rules/...`
4. 替换后 YAML 语法验证通过
5. `check_urls.py` 已删除
6. 不要有遗漏——每个文件单独检视确认

## 测试要求

变更完成后执行：

```bash
python -c "
import yaml
for f in ['mihomo/config/ProMax.yaml', 'mihomo/config/Pro.yaml', 'mihomo/config/Standard.yaml']:
    yaml.safe_load(open(f, 'r', encoding='utf-8'))
    print(f)
print('All YAML valid')
"

# 验证无残留被墙地址
grep -n 'google\.com' mihomo/config/*.yaml
grep -n 'gstatic\.com' mihomo/config/*.yaml
grep -n 'raw\.githubusercontent\.com/huyanluanyuqaq' mihomo/config/*.yaml
# 以上三条应无输出
```

## 风险说明

- jsDelivr CDN 在国内有时也会间歇性被干扰，但整体可用性远高于 `raw.githubusercontent.com`
- `cp.cloudflare.com` 在国内节点策略可能随时间变化，但优于已确认被封锁的 `google.com`
- 规则集源变更后，已有缓存的规则不受影响，新拉取走 CDN
- 本变更不涉及代理工作方式的改变，只改进首次启动的拉取成功率
