# P016: 仓库审查修复（6 项）

## 任务目标

根据仓库审查报告，修复以下 6 个问题：

1. Pro.yaml 删除 YouTube 相关规则（目标组不存在）
2. Standard.yaml 补充 proxy-groups 和 dns 模块
3. 统一 IP 规则集路径为 `ipcidr`
4. 启用 ProMax.yaml 中未使用的 `&hk-first` 锚点
5. Pro.yaml 删除未使用的 rule-provider（Netflix/Spotify/Disney）

## 修改范围

| 文件 | 操作 |
|---|---|
| `mihomo/config/Pro.yaml` | 删除 YouTube 规则 + IP 路径修正 + 删除未使用 provider |
| `mihomo/config/Standard.yaml` | 补充 proxy-groups + dns + IP 路径修正 |
| `mihomo/config/ProMax.yaml` | 启用 hk-first 锚点 + IP 路径修正 |

## 详细修改

### 修改 1：Pro.yaml — 删除 YouTube 规则

删除以下两行：

```diff
-  - RULE-SET,YouTube,Youtube
-  - RULE-SET,YoutubeIP,Youtube,no-resolve
```

同时删除对应的 rule-provider：

```diff
-  YouTube:    { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/YouTube.mrs, proxy: China }
-  YoutubeIP:    { <<: *rule-set-ipcidr, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/ip/YoutubeIP.mrs, proxy: China }
```

### 修改 2：Standard.yaml — 补充 proxy-groups

参照 Pro.yaml 的 proxy-groups 结构（Pro.yaml 是精简版但功能完整），为标准版补充以下策略组：

- `Telegram` — select, `*sg-first`
- `Ai` — select, `*us-first`
- `Socialmedia` — select, `*us-first`
- `Develop` — select, `*us-first`
- `Streaming` — select, `*us-first`
- `Apple` — select, `*us-first`
- `Proxy` — select, `*oversea`
- `China` — select, `*direct`
- `Final` — select, `*oversea`
- `故障转移` — fallback, 同 Pro.yaml
- `节点选择` — select, include-all: true
- 6 个区域 url-test 节点组（香港/台湾/日本/狮城/韩国/美国），同 Pro.yaml

**注意**：Standard.yaml 的 proxy-groups 结构与 Pro.yaml 一致（url-test 直接作为区域组，无 select 包装层，无负载均衡组）。

补充 dns 模块，内容与 Pro.yaml 一致：

```yaml
dns:
  enable: true
  ipv6: false
  enhanced-mode: fake-ip
  fake-ip-range: 198.18.0.1/16
  default-nameserver: [119.29.29.29, 180.184.1.1, 223.5.5.5]
```

### 修改 4：ProMax.yaml — 启用 hk-first 锚点

`&hk-first` 锚点定义为：

```yaml
[hong_kong_nodes, 故障转移, 节点选择, taiwan_node, japan_node, singapore_node, korea_node, us_node]
```

将其用于 `Telegram` 策略组（Telegram 属于亚洲优先场景），替换原来的 `*sg-first`：

```diff
-  - { name: Telegram, type: select, proxies: *sg-first, icon: ... }
+  - { name: Telegram, type: select, proxies: *hk-first, icon: ... }
```

> 说明：Telegram 在亚洲用户使用较多，香港节点优先更合理。如用户有不同偏好可调整。

### 修改 5：Pro.yaml — 删除未使用的 rule-provider

删除以下从未在 `rules` 段中被引用的 rule-provider：

```diff
-  Netflix:    { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Netflix.mrs, proxy: China }
-  NetflixIP:    { <<: *rule-set-ipcidr, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/ip/NetflixIP.mrs, proxy: China }
-  Spotify:    { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Spotify.mrs, proxy: China }
-  Disney:    { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Disney.mrs, proxy: China }
```

> 注意：NetflixIP 的 IP 路径同时也需要修正（修改 3），但因为 Netflix/NetflixIP 整体要删除，所以直接删除即可，不需要单独修正。
