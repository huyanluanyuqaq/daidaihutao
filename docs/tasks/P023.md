# 任务 P023：新增 Steam 规则源

## 描述

在 `mihomo/config/ProMax.yaml` 中新增 Steam 规则支持：
- 新增 `Steam` 策略组，置于 `Speedtest` 下方，规则引用 `Steam.mrs`，策略组使用 `Steam`
- 新增规则引用 `Steam-cn.mrs`，置于 `Cusdirect` 下方，策略组使用 `China`

## 背景

上游规则集新增 Steam 相关规则文件：
- `Steam.mrs`：用于匹配 Steam 相关域名，需要走 Steam 策略组
- `Steam-cn.mrs`：用于匹配 Steam 国内相关规则，需要走 China 策略组

需要在 ProMax 配置中接入这两个规则源，确保规则优先级与现有策略组结构一致。

## 修改内容

### 文件列表

| 文件 | 变更类型 |
|---|---|
| `mihomo/config/ProMax.yaml` | 新增 policy group、rules 条目、rule-providers 条目 |

### 具体变更

| # | 位置 | 操作 | 说明 |
|---|---|---|---|
| 1 | `proxy-groups` 段 | 新增 | 添加 `Steam` 策略组，置于 `Speedtest` 下方，使用 `*oversea` 代理列表 |
| 2 | `rules` 段 #5 特殊垂直领域 | 新增 | 添加 `RULE-SET,Steam,Steam`，置于 `Speedtest` 下方 |
| 3 | `rules` 段 #3 高优先级特定服务 | 新增 | 添加 `RULE-SET,Steam-cn,China`，置于 `Cusdirect` 下方 |
| 4 | `rule-providers` 段 | 新增 | 添加 `Steam` 规则源，引用 `Steam.mrs` |
| 5 | `rule-providers` 段 | 新增 | 添加 `Steam-cn` 规则源，引用 `Steam-cn.mrs` |

### 修改后效果

```
修改前：
  - { name: Speedtest, type: select, proxies: *oversea, icon: ... }
  ...
  - RULE-SET,Cusdirect,Cusdirect
  - RULE-SET,Emby,EMBY
  ...
  - RULE-SET,Speedtest,Speedtest
  ...
  Speedtest:      { <<: *rule-set-domain, url: .../Speedtest.mrs }

修改后：
  - { name: Speedtest, type: select, proxies: *oversea, icon: ... }
  - { name: Steam, type: select, proxies: *oversea, icon: ... }
  ...
  - RULE-SET,Cusdirect,Cusdirect
  - RULE-SET,Steam-cn,China
  - RULE-SET,Emby,EMBY
  ...
  - RULE-SET,Speedtest,Speedtest
  - RULE-SET,Steam,Steam
  ...
  Speedtest:      { <<: *rule-set-domain, url: .../Speedtest.mrs }
  Steam:          { <<: *rule-set-domain, url: .../Steam.mrs }
  Steam-cn:       { <<: *rule-set-domain, url: .../Steam-cn.mrs }
```

## 注意事项

1. 仅修改 `ProMax.yaml`
2. `Steam` 策略组置于 `Speedtest` 下方，使用与 `Speedtest` 相同的 `*oversea` 代理列表
3. `Steam-cn` 规则置于 `Cusdirect` 下方，策略组为 `China`
4. `rule-providers` 段新增 `Steam` 与 `Steam-cn` 规则源，保持 YAML 缩进一致
