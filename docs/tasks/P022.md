# 任务 P022：新增 Selfdirect 规则源

## 描述

在 `mihomo/config/ProMax.yaml` 中新增规则源 `Selfdirect`，该规则源来自上游仓库 `huyanluanyuqaq/hutaorules` 的 `Selfdirect.mrs`，在规则排序中排在 `Embydirect` 上方，使用 `DIRECT` 策略直连。

## 背景

上游仓库新增了 `Selfdirect.mrs` 规则集，用于匹配自建服务/自管设备的域名（如 NAS、家庭服务器、内网穿透服务等）。这些流量需要直接连接（DIRECT），不应走代理。需要将其接入 ProMax 配置，确保匹配优先级高于 Embydirect 等流媒体直连规则。

## 修改内容

### 文件列表

| 文件 | 变更类型 |
|---|---|
| `mihomo/config/ProMax.yaml` | 新增 rule-provider、rules 条目、fake-ip-filter 条目 |

### 具体变更

| # | 位置 | 操作 | 说明 |
|---|---|---|---|
| 1 | `rule-providers` 段 | 新增 | 添加 `Selfdirect` 规则源，从上游仓库拉取 `Selfdirect.mrs`（domain 类型） |
| 2 | `rules` 段 #3 高优先级特定服务 | 新增（排于 Embydirect 上方） | 添加 `RULE-SET,Selfdirect,DIRECT`，使自建服务流量优先直连 |
| 3 | `fake-ip-filter` 段 | 新增（排于 Embydirect 上方） | 添加 `RULE-SET,Selfdirect,real-ip`，确保自建服务域名获取真实 IP |

### 修改后效果

```
修改前：
# 3. 高优先级特定服务
  - RULE-SET,Ai-cn,China
  - RULE-SET,Embydirect,Embydirect     ← 当前第一条
  - RULE-SET,Cusdirect,Cusdirect
  ...

修改后：
# 3. 高优先级特定服务
  - RULE-SET,Ai-cn,China
  - RULE-SET,Selfdirect,DIRECT          ← 新增，排在 Embydirect 上方
  - RULE-SET,Embydirect,Embydirect
  - RULE-SET,Cusdirect,Cusdirect
  ...
```

### 涉及配置段

#### 1. fake-ip-filter（DNS 真实 IP 解析）

`Selfdirect` 域名需要获取真实 IP（不走 fake-ip），因为在直连场景下 fake-ip 会导致连接失败。

```yaml
# 插入位置：Embydirect 上方
    - RULE-SET,Selfdirect,real-ip
```

#### 2. proxy-groups（策略组）

`Selfdirect` 使用 DIRECT 内置策略，无需新增专用策略组。

#### 3. rules（规则路由）

```yaml
# 3. 高优先级特定服务
  - RULE-SET,Ai-cn,China
  - RULE-SET,Selfdirect,DIRECT     # 新增
  - RULE-SET,Embydirect,Embydirect
  ...
```

#### 4. rule-providers（规则集提供者）

```yaml
  Selfdirect:     { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Selfdirect.mrs }
```

## 注意事项

1. 仅在 `ProMax.yaml` 中变更——Pro 和 Standard 配置不涉及此规则源
2. `Selfdirect` 使用 DIRECT 策略直连，无需新增代理组
3. 确保 `fake-ip-filter` 中 `RULE-SET,Selfdirect,real-ip` 的插入顺序与 `rules` 段一致
4. `rule-providers` 段新增条目按字母顺序插入适当位置（S 开头的 `Selfdirect` 应排在 `Socialmedia` 之后、`Speedtest` 之前）
5. 保持 YAML 缩进一致（2 空格）
