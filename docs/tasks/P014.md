# P014: 修复 QUIC/UDP 管控 AND 规则 — 位置重排 + GEOIP 替换为 RULE-SET

## 任务目标

1. 将 `# 7. QUIC / UDP 443 管控` 下的 AND 规则移到 `TrackingIP` 下方（广告拦截区域）
2. 将 AND 规则中的 `GEOIP,CN,no-resolve` 替换为 `RULE-SET,ChinaIP`，与全仓库 rule-set 风格保持一致

## 修改范围

| 文件 | 操作 |
|---|---|
| `mihomo/config/ProMax.yaml` | rules 段：挪动 AND 规则 + 替换条件表达式 |

## 修改前

```yaml
rules:
  # 2. 广告拦截
  - RULE-SET,Tracking,Tracking
  - RULE-SET,Advertising,Tracking
  - RULE-SET,AdvertisingIP,Tracking,no-resolve
  - RULE-SET,TrackingIP,Tracking,no-resolve
  ...
  # 7. QUIC / UDP 443 管控
  - AND,((DST-PORT,443),(NETWORK,UDP),(NOT,((GEOIP,CN,no-resolve)))),Tracking
```

## 修改后

```yaml
rules:
  # 2. 广告拦截
  - RULE-SET,Tracking,Tracking
  - RULE-SET,Advertising,Tracking
  - RULE-SET,AdvertisingIP,Tracking,no-resolve
  - RULE-SET,TrackingIP,Tracking,no-resolve
  - AND,((DST-PORT,443),(NETWORK,UDP),(NOT,((RULE-SET,ChinaIP)))),Tracking
  ...
  #（移除原有的 # 7. QUIC / UDP 443 管控 独立区域）
```

## 变更明细

### 变更 1：位置调整

- **移动**：将 AND 规则从原 `# 7. QUIC / UDP 443 管控` 区域移到 `# 2. 广告拦截` 区域，放在 `TrackingIP` 规则正下方
- **移除**：删除 `# 7. QUIC / UDP 443 管控` 注释行（该区域仅剩此一条规则）
- 其他分区编号保持不变

### 变更 2：GEOIP → RULE-SET

| 字段 | 当前值 | 目标值 |
|---|---|---|
| AND 条件中的 IP 匹配 | `GEOIP,CN,no-resolve` | `RULE-SET,ChinaIP` |

- `ChinaIP` rule-provider 已在 `rule-providers` 段定义（第 251 行），可直接引用
- 替换后全配置文件的 rules 段统一使用 `RULE-SET`，无 `GEOIP` 残留
