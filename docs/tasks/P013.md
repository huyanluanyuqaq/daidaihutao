## 变更 2：路由策略组重排

将 `proxy-groups → # 路由策略` 区域按 rules 的分组顺序重新排列，并添加分组注释。

### 当前顺序

```
# 路由策略（当前无序）
Tracking, Speedtest, Telegram, Socialmedia, Ai, Develop, EMBY,
Embydirect, Streaming, Crypto, Youtube, Google, Microsoft, Apple,
Proxy, China, Final
```

### 目标顺序（对应 rules 分组）

```yaml
  # 路由策略
  # 2. 广告拦截
  - { name: Tracking, ... }

  # 3. 高优先级特定服务
  - { name: Telegram, ... }
  - { name: Ai, ... }

  # 4. 专属流媒体与社交
  - { name: Youtube, ... }
  - { name: Streaming, ... }
  - { name: Embydirect, ... }
  - { name: EMBY, ... }
  - { name: Socialmedia, ... }

  # 5. 特殊垂直领域
  - { name: Crypto, ... }
  - { name: Develop, ... }
  - { name: Speedtest, ... }

  # 6. 大厂特定服务
  - { name: Google, ... }
  - { name: Microsoft, ... }
  - { name: Apple, ... }

  # 8. 通用国外流量
  - { name: Proxy, ... }

  # 9. 通用国内流量
  - { name: China, ... }

  # 10. 兜底规则
  - { name: Final, ... }
```

**注意**：组号跳过 #1（本地/私有无对应策略组）和 #7（QUIC 管控无对应策略组）。

策略组内容（proxies、icon 等字段）保持不变，仅调整顺序和添加分组注释。`# 自动选择` / `# 策略组区域` 等基础设施组（故障转移、节点选择、香港节点等）**保持原位不动**。
