# P009: ProMax.yaml 规则排序同步到所有配置文件

## 任务目标

将 ProMax.yaml 中 rules（规则路由）的标准分组排序（共 10 组，编号 #1~#10）同步到 `Pro.yaml` 和 `Standard.yaml`，使所有配置文件的 rules 段使用统一的分组结构和编号体系。

## ProMax.yaml 标准排序（参考基准）

```
# 1.  本地/私有           → Private, PrivateIP
# 2.  广告拦截             → Tracking, Advertising, AdvertisingIP, TrackingIP
# 3.  高优先级特定服务      → Embydirect, Emby, Ai, AiIP, Telegram, TelegramIP
# 4.  专属流媒体与社交      → Youtube, YoutubeIP, Streaming, StreamingIP, Socialmedia, SocialmediaIP
# 5.  特殊垂直领域          → Crypto, Dev, Speedtest
# 6.  大厂特定服务          → Google, GoogleIP, Applecn, Apple, AppleIP, Microsoft
# 7.  QUIC / UDP 443 管控  → AND((DST-PORT,443),(NETWORK,UDP),(NOT,((GEOIP,CN,no-resolve))))
# 8.  通用国外流量          → Proxy, ProxyIP
# 9.  通用国内流量          → China, ChinaIP
# 10. 兜底规则              → MATCH
```

## 修改范围

| 文件 | 操作 |
|---|---|
| `mihomo/config/Pro.yaml` | 重排 rules + rule-providers（保留现有规则，不增删） |
| `mihomo/config/Standard.yaml` | 重排 rules + rule-providers（保留现有规则，不增删） |

## 禁止修改项

- 不修改 `ProMax.yaml`
- 不修改 `mihomo/openclash/` 下的 `.conf` 文件
- 不修改 `docs/`、`PROJECT_LESSONS.md`、`PROJECT-SPEC.md`
- 不增删任何规则，只调整排序和分组注释
- 不修改规则集提供者的 URL 或配置参数

## 详细变更

### 变更 1：Pro.yaml — rules 重排

**目标排序**（对应 ProMax 组号）：

```
# 1. 本地/私有          → Private, PrivateIP
# 2. 高优先级特定服务     → TM, Telegram, TelegramIP, AI, AiIP, Dev
# 3. 专属流媒体与社交     → YouTube, YoutubeIP, Streaming, StreamingIP, SocialMedia, SocialMediaIP
# 4. 大厂特定服务        → Apple
# 5. 通用国外流量        → Proxy, ProxyIP
# 6. 通用国内流量        → China, ChinaIP
# 7. 兜底规则            → MATCH
```

Pro.yaml 不包含广告拦截（#2）、特殊垂直领域（#5）、QUIC 管控（#7）和部分大厂规则，所以跳过对应组号。规则内容 **一条不增、一条不减**。

**当前排序**（变更前）：

```
# 本地/私有 → Private, PrivateIP
# 即时通讯 → TM, Telegram, TelegramIP
# 社交平台 → SocialMedia, SocialMediaIP
# 人工智能 → AI, AiIP
# 开发服务 → Dev
# 流媒体 → Streaming, StreamingIP
# Apple → Apple
# Youtube → YouTube, YoutubeIP
# 国外流量 → Proxy, ProxyIP
# 国内流量 → China, ChinaIP
# 兜底 → MATCH
```

**主要变化**：

- YouTube/YoutubeIP 从独立段归入「专属流媒体与社交」组
- SocialMedia/SocialMediaIP 从独立段归入「专属流媒体与社交」组
- TM/Telegram/TelegramIP 改为「高优先级特定服务」
- Dev 改为「高优先级特定服务」
- 所有分组注释添加编号（#1~#7）
- 组间添加空行分隔

### 变更 2：Pro.yaml — rule-providers 重排

按 domain → ipcidr 分组，每组内按规则匹配顺序排序：

```
# domain
Private → TM → Telegram → AI → Dev → YouTube → Streaming → Netflix → Spotify → Disney → SocialMedia → Apple → Proxy → China
# ipcidr
PrivateIP → TelegramIP → SocialMediaIP → AiIP → NetflixIP → StreamingIP → ProxyIP → ChinaIP → YoutubeIP
```

**注意**：
- Netflix、Spotify、Disney 虽然在 rules 段未引用，但在 rule-providers 中存在，保留不动
- YoutubeIP 在变更前缺少，需补回

### 变更 3：Standard.yaml — rules 重排

**目标排序**：

```
# 1. 本地/私有    → Private, PrivateIP
# 8. 通用国外流量  → Proxy, ProxyIP
# 9. 通用国内流量  → China, ChinaIP
# 10. 兜底规则    → MATCH
```

Standard.yaml 只有 4 组，组号与 ProMax 保持一致，跳过中间组号。

**当前排序**（变更前）：

```
# 本地/私有 → Private, PrivateIP
# 国外流量 → Proxy, ProxyIP
# 国内流量 → China, ChinaIP
# 兜底 → MATCH
```

**主要变化**：
- 组间添加空行分隔
- 添加编号注释

### 变更 4：Standard.yaml — rule-providers 重排

```
# domain
Private → Proxy → China
# ipcidr
PrivateIP → ProxyIP → ChinaIP
```

## 验收标准

1. Pro.yaml 的 rules 段按变更 1 目标排序排列，组号连续 #1~#7
2. Pro.yaml rule-providers 按 domain → ipcidr 分组，顺序匹配 rules
3. Standard.yaml 的 rules 段按变更 3 目标排序排列，组号为 #1/#8/#9/#10
4. Standard.yaml rule-providers 按 domain → ipcidr 分组
5. ProMax.yaml 内容完全不变
6. 不增删任何规则或 rule-provider（YoutubeIP 补回除外，属修复遗漏）
7. OpenClash `.conf` 文件不变
8. 每组规则前有明确的 #编号 注释
