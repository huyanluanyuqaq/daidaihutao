# P002: Pro.yaml 规则集提供者引用修复

## 任务目标

根据规则仓库 `huyanluanyuqaq/hutaorules`（`rules` 分支）的实际文件清单，修复 `mihomo/config/cn/Pro.yaml` 中 `rule-providers` 和 `rules` 部分，确保所有引用的规则文件在远程仓库真实存在且路径正确。

## 背景

当前 `Pro.yaml` 存在三类问题：

| 问题 | 说明 | 数量 |
|---|---|---|
| 🔴 URL 路径错误 | IP 类规则集引用 `mihomo/ip/`，实际应为 `mihomo/ipcidr/` | 全部 IP 规则 |
| 🔴 引用不存在文件 | 上游仓库中这些规则已被删除 / 从未生成 | 8 条 |
| 🟡 文件名大小写不匹配 | 仓库实际文件名为首字母小写，配置使用了首字母大写 | 4 条（含 1 个 IP） |
| 🟢 可新增规则 | 仓库中有但 Pro.yaml 未使用的规则集 | 6 条 |

## 修改内容

### 修改一：删除不存在于仓库的规则集（8 条）

以下规则集在仓库中不存在，从 `rule-providers` 和 `rules` 中同时移除：

| rule-providers | URL 对应文件 | 对应的 rules 行 |
|---|---|---|
| TM | TM.mrs | `- RULE-SET,TM,即时通讯` |
| Netflix | Netflix.mrs | `- RULE-SET,Netflix,国际媒体` |
| Spotify | Spotify.mrs | `- RULE-SET,Spotify,国际媒体` |
| Disney | Disney.mrs | `- RULE-SET,Disney,国际媒体` |
| Games | Games.mrs | `- RULE-SET,Games,游戏平台` |
| Facebook | Facebook.mrs | `- RULE-SET,Facebook,脸书服务` |
| NetflixIP | NetflixIP.mrs | `- RULE-SET,NetflixIP,国际媒体,no-resolve` |
| EmbyIP | EmbyIP.mrs | `- RULE-SET,EmbyIP,EMBY,no-resolve` |

> **影响**：删除后相关域名 / IP 走兜底的 `国外流量` 策略组（TM、Netflix、Spotify、Disney、Games、Facebook 域名经 Proxy 匹配后将命中 `Proxy → 国外流量`），不会漏泄。

### 修改二：修正文件名大小写对齐仓库（4 条）

规则提供者名称 key、rules 中的引用名、URL 中的文件名 **全部对齐仓库实际文件名**：

| 原规则提供者名 | 原 rules 中的名称 | 原 URL 文件名 | → 改为 |
|---|---|---|---|
| SocialMedia | SocialMedia | SocialMedia.mrs | **Socialmedia** |
| SocialMediaIP | SocialMediaIP | SocialMediaIP.mrs | **SocialmediaIP** |
| YouTube | （无对应 rule） | YouTube.mrs | **Youtube** |
| AI | AI | AI.mrs | **Ai** |

> 注意：YouTube 在 Pro.yaml 的 `rule-providers` 中有声明但 `rules` 中未引用，仅修改 provider key 和 URL 即可，无需改 rules。

### 修改四：修复 IP 类 URL 路径（涉及全部 IP 规则集）

将所有 behavior: `ipcidr` 的规则集 URL 路径从：
```
https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/ip/Xxx.mrs
```
改为：
```
https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/ipcidr/Xxx.mrs
```

涉及全部共 10 个 IP 规则集：
PrivateIP / TelegramIP / SocialmediaIP（原名 SocialMediaIP）/ AiIP / EmbyIP → 删除 / NetflixIP → 删除 / StreamingIP / ProxyIP / ChinaIP / GoogleIP

以及新增的 4 个 IP 规则集（AppleIP / AdvertisingIP / TrackingIP / YoutubeIP）需直接使用 `ipcidr/` 路径。

## 禁止修改项

- 不修改 `proxy-groups`（策略组）部分
- 不修改 Lite.yaml 和 Mini.yaml
- 不修改 `rules/` 目录下的规则文件
- 不修改 `proxy-providers` 配置
- 不修改 Pro.yaml 的基础设置（端口、TUN、DNS 等）
- 不修改 rules 中已有规则的策略组分配（只修改规则名）
- 不做额外的重构或格式化

## 验收标准

1. Pro.yaml 中所有 `rule-providers` 条目指向的 `.mrs` 文件在仓库中**真实存在**
2. 所有 IP 类规则集 URL 使用正确的 `mihomo/ipcidr/` 路径
3. 所有规则文件名大小写与仓库实际文件名**完全一致**
4. 所有删除的规则集对应 `rules` 中的 RULE-SET 行也已同步删除
5. 新增的 6 个规则集在 `rule-providers` 和 `rules` 中均有对应声明
6. YAML 语法验证通过
7. Pro.yaml 最终 provider 数 = 19（domain）+ 12（ipcidr）= 31 个（匹配仓库）

## 风险说明

- **低风险**：删除的 8 个规则集对应内容较为冷门或已不存在，移除后相关流量走兜底的策略组，不会断网
- **规则名重命名风险**：Socialmedia、Ai、Youtube 等名称变更可能导致短时策略行为差异，但语义不变
- **新增规则风险**：新增 6 个规则集使匹配更精确，无副作用
- **中等风险**：YAML 格式错误（缩进、冒号缺失等）可能导致 mihomo 启动失败，修改后必须 YAML 验证
- 建议修改完成后在本地 mihomo 客户端加载测试
