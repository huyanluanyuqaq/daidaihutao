# P004: 简化 Claude 允许修改范围为 docs/ 目录

## 任务目标

当前 Claude 的允许修改范围（`docs/`、`rules/`、`mihomo/config/`）边界模糊，是 Claude 越权执行 Agent 工作的根本原因。需要将 Claude 的允许修改范围严格限定为 `docs/` 目录，删除宽松条款和执行例外。

## 修改范围

修改 `PROJECT-SPEC.md` 中的"Agent 分工"章节：

### 1. Claude 职责调整（第 48-82 行）

**当前：**
```
允许修改：
docs/
rules/
mihomo/config/

允许场景：
规范维护
Review 修复
示例实现
Hotfix
```

**改后：**
```
允许修改：
docs/

允许场景：
规范维护
Review 文档编写
```

改动要点：
- 删除 `rules/` 和 `mihomo/config/` 允许目录
- 删除"Review 修复""示例实现""Hotfix"三个允许场景——这些是执行 Agent 的工作
- "Review 文档编写"已包含在 Claude 的职责（编写 Review 文件）中，写出来更清晰

### 2. 执行 Agent 允许范围调整

Codex、MiMoCode、DeveCo 三者的"允许修改"中补充 `docs/` 之外的目录：

**当前：**
```

rules/
tests/        如存在
src/          如存在
```

**改后：**
```
允许修改：
除 docs/ 外的所有项目目录
```

注意：执行 Agent 的"默认禁止修改"中 `docs/reviews/`、`docs/archive/`、`PROJECT-SPEC.md`、`Review 结论` 保留不变。

### 3. 同步更新"核心原则"（第 225-237 行）

将"核心原则"中的原则 11 补充或精炼，强调：
> 所有非 docs/ 的修改必须由执行 Agent 完成，Claude 不直接修改 docs/ 以外的文件

## 禁止修改项

- 不得修改 Claude 的职责描述（制定规范、拆解任务、编写任务文档、审查执行结果等）
- 不得修改 4 Agent 的职责分配和分支模型
- 不得修改三阶段任务流程结构
- 不得修改提交规范表
- 不得修改 PROJECT-SPEC.md 的其他章节（Agent 分工章节之外）

## 验收标准

1. Claude 的允许修改目录只剩 `docs/`（删掉 `rules/` 和 `mihomo/config/`）
2. Claude 的允许场景只剩"规范维护"和"Review 文档编写"（删掉"Review 修复""示例实现""Hotfix"）
3. 三个执行 Agent 的允许修改统一改为"除 docs/ 外的所有项目目录"
4. 执行 Agent 的禁止修改（docs/reviews/、docs/archive/、PROJECT-SPEC.md、Review 结论）保持不变
5. 核心原则中体现"Claude 不直接修改 docs/ 以外的文件"

## 测试要求

- PROJECT-SPEC.md 格式不变，仍是有效 markdown
- 章节标题、列表缩进保持一致

## 风险说明

- 此变更生效后，Claude 遇到 docs/ 之外的任何文件变更需求（包括 README.md 根目录），都必须走任务文档 → 执行 Agent 的完整流程
- 无功能影响，仅流程变更
