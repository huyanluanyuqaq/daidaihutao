# 任务 P016：DNS 增强、fake-ip-filter 规则模式与自定义策略组

## 描述

修改 `mihomo/config/ProMax.yaml` 和 `mihomo/config/Pro.yaml`，完成 DNS 模块增强、fake-ip-filter 规则模式配置，以及新增自定义策略组。Standard.yaml 仅同步 DNS 监听端口变更。

## 修改内容

### 1. DNS 模块添加监听端口 7874

**适用文件：** `mihomo/config/ProMax.yaml`、`mihomo/config/Pro.yaml`、`mihomo/config/Standard.yaml`

在 dns 区域 `enable: true` 之后添加：
```yaml
listen: 7874
```

变更位置示例（以 ProMax.yaml 为准，其他文件同理）：
- 原值：
  ```yaml
  dns:
    enable: true
    ipv6: false
  ```
- 目标值：
  ```yaml
  dns:
    enable: true
    listen: 7874
    ipv6: false
  ```

### 2. DNS fake-ip-filter-mode: rule 及提取直连 rule-providers 加入 fake-ip-filter

**适用文件：** `mihomo/config/ProMax.yaml`、`mihomo/config/Pro.yaml`

#### 2a. 添加 `fake-ip-filter-mode: rule`

在 `enhanced-mode: fake-ip` 之后添加 `fake-ip-filter-mode: rule`：

- 原值：
  ```yaml
  enhanced-mode: fake-ip
  fake-ip-range: 198.18.0.1/16
  ```
- 目标值：
  ```yaml
  enhanced-mode: fake-ip
  fake-ip-filter-mode: rule
  fake-ip-range: 198.18.0.1/16
  ```

#### 2b. 将直连 rule-providers 加入 fake-ip-filter

**ProMax.yaml** 直连 rule-providers（规则路由中指向 DIRECT 或使用 `*direct` 锚点的策略组所引用的 domain 类 rule-provider）：

| rule-provider | 规则引用 | 指向策略组 | 判定依据 |
|---|---|---|---|
| Private | `RULE-SET,Private,DIRECT` | DIRECT | 直连 |
| Applecn | `RULE-SET,Applecn,China` | China（`*direct`） | 直连 |
| Embydirect | `RULE-SET,Embydirect,Embydirect` | Embydirect（`*direct`） | 直连 |
| China | `RULE-SET,China,China` | China（`*direct`） | 直连 |

ProMax.yaml fake-ip-filter 变更：
- 原值：
  ```yaml
  fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com]
  ```
- 目标值：
  ```yaml
  fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com, 'rule-set:Private', 'rule-set:Applecn', 'rule-set:Embydirect', 'rule-set:China']
  ```

**Pro.yaml** 直连 rule-providers：

| rule-provider | 规则引用 | 指向策略组 | 判定依据 |
|---|---|---|---|
| Private | `RULE-SET,Private,DIRECT` | DIRECT | 直连 |
| China | `RULE-SET,China,China` | China（`*direct`） | 直连 |

Pro.yaml fake-ip-filter 变更：
- 原值：
  ```yaml
  fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com]
  ```
- 目标值：
  ```yaml
  fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com, 'rule-set:Private', 'rule-set:China']
  ```

**Standard.yaml** 同理，直连 rule-providers 为 Private 和 China：
- 目标值：
  ```yaml
  fake-ip-filter: [+.miwifi.com, +.docker.io, +.market.xiaomi.com, +.push.apple.com, 'rule-set:Private', 'rule-set:China']
  ```

Standard.yaml 也需同步添加 `fake-ip-filter-mode: rule`，位置同上。

### 3. 新建 Cusproxy 和 Cusdirect 策略组

**适用文件：** `mihomo/config/ProMax.yaml`、`mihomo/config/Pro.yaml`

#### 3a. 策略组定义

在 proxy-groups 区域的路由策略部分，Ai 策略组**之前**插入以下两个策略组：

```yaml
  - { name: Cusproxy, type: select, proxies: *oversea, icon: https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/Global.png }
  - { name: Cusdirect, type: select, proxies: *direct, icon: https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/China.png }
```

- `Cusproxy` 使用 proxy 图标（Global.png），proxies 引用 `*oversea`
- `Cusdirect` 使用 china 图标（China.png），proxies 引用 `*direct`

#### 3b. 规则引用

在 rules 区域，**Ai 规则之前**（即高优先级特定服务区域的最前面）插入：

**ProMax.yaml：**
```yaml
  # 自定义策略
  - RULE-SET,Cusproxy,Cusproxy
  - RULE-SET,Cusdirect,Cusdirect
```

**Pro.yaml：**
```yaml
  # 自定义策略
  - RULE-SET,Cusproxy,Cusproxy
  - RULE-SET,Cusdirect,Cusdirect
```

规则优先级：Cusproxy/Cusdirect 规则位于 Ai 规则之上，确保自定义策略优先于 AI 策略。

#### 3c. rule-providers 定义

在 rule-providers 区域添加：

```yaml
  Cusproxy:    { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Cusproxy.mrs, proxy: China }
  Cusdirect:   { <<: *rule-set-domain, url: https://raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/mihomo/domain/Cusdirect.mrs, proxy: China }
```

## 变更总结

| 文件 | 变更项 |
|---|---|
| ProMax.yaml | dns.listen=7874, fake-ip-filter-mode=rule, fake-ip-filter 增加直连 rule-set 引用, 新增 Cusproxy/Cusdirect 策略组+规则+providers |
| Pro.yaml | dns.listen=7874, fake-ip-filter-mode=rule, fake-ip-filter 增加直连 rule-set 引用, 新增 Cusproxy/Cusdirect 策略组+规则+providers |
| Standard.yaml | dns.listen=7874, fake-ip-filter-mode=rule, fake-ip-filter 增加直连 rule-set 引用 |

## 注意事项

1. `fake-ip-filter-mode: rule` 是 mihomo Meta 的特性，配合 `rule-set:` 前缀的 fake-ip-filter 条目使用，使 fake-ip 过滤基于规则集动态判断而非静态域名列表
2. `rule-set:` 前缀引用的是 rule-providers 中已定义的 domain 行为类型的规则集名称
3. 仅提取 domain 行为的直连 rule-providers，ipcidr 行为的不纳入 fake-ip-filter（fake-ip-filter 仅处理域名）
4. Cusproxy/Cusdirect 的 rule-providers URL 遵循项目已有的 hutaorules 仓库路径规范，实际规则文件需由外部规则构建流程生成推送
