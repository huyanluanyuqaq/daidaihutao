# P005: 优化 AGENTS.md 和 CLAUDE.md 工作原则 + 完善远程推送流程

## 任务目标

1. 优化 `AGENTS.md` 和 `CLAUDE.md` 的工作原则，两个文件保持一致
2. 将"工作分支提交后先推送到对应远程分支，再合并 main，再推 main 远程"这一环节明确加入协作流程描述
3. 更新 `PROJECT-SPEC.md` 的协作流程章节，补充各阶段对应的远程推送步骤

## 修改范围

### 1. CLAUDE.md — 工作原则优化

**当前内容**（需要替换）：
```
- 修改前先简要说明计划，然后直接执行。
- 优先做最小必要修改，不做无关重构。
- 不删除用户已有内容，除非任务明确要求。
- 不引入不必要的新依赖。
```

**改后内容**：
```
- Claude 的修改权限仅限 docs/ 目录，其他目录变更必须通过任务文档走执行 Agent 流程。
- 创建任务文档并合并到 main 后，等待用户指令再执行下一步。
- 所有 commit 必须使用 [Agent][Pxxx] 格式前缀。
- Review 结论只能是 PASS 或 FAIL，必须附带逐项验证清单。
- 如存在重大歧义或高风险操作，再向用户确认。
- 优先做最小必要修改，不做无关重构。
- 所有输出尽量清晰、可复查、可复用。
```

删除 `"修改前先简要说明计划，然后直接执行"` － 与 docs-only 分工冲突。
删除 `"不删除用户已有内容"` 和 `"不引入不必要的新依赖"` － 已隐含在流程中，不再需要单独列出。

### 2. AGENTS.md — 新增/重写为协作速查手册

结构如下：
- **# Agent 协作分工** — 标题
- **## 角色定位** — 4 Agent 角色表（含分支、允许修改目录、职责一句话）
- **## 完整工作流**（含远程推送）
- **## 提交前缀** — 4 Agent 提交格式表
- **## 核心规则** — 关键原则

**工作流描述中必须体现：**

```
阶段一（Claude 规划）：
  claude/review 创建任务文档 → 合并到 main → 推送 origin/main → 等待指令

阶段二（执行 Agent，以 Codex 为例）：
  codex/automate 执行修改 → 提交 → 推送 origin/codex/automate → 合并到 main → 推送 origin/main → 等待 Review

阶段三（Claude Review）：
  claude/review 写 Review 文件 → 合并到 main → 推送 origin/main → PASS 后归档 → 再推送 origin/main
```

关键新增：**每个阶段结束都推送远程分支**，确保工作进度不丢失。
