# P007: 修正 AGENTS.md 中模糊的"同步"描述

## 任务目标

修正 `AGENTS.md` 中 `阶段二` 和 `阶段三` 关于工作分支同步流程的描述，将模糊的"从 main 同步"替换为精确的 git 操作序列，防止执行 Agent 误执行 `git merge --ff-only origin/main` 等违反协作流程的命令。

## 修改范围

**仅修改以下文件：**

- `AGENTS.md` — 项目根目录

### 具体修改内容

#### 修改 1：阶段二（第 29 行附近）

**原内容：**

```
codex/automate 从 main 同步，读取 docs/tasks/Pxxx.md
```

**修改为：**

```
codex/automate 先更新 main（git fetch origin + git pull --ff-only origin main），再 rebase main（git rebase main），然后读取 docs/tasks/Pxxx.md
```

#### 修改 2：阶段三（第 44 行附近）

**原内容：**

```
claude/review 从 main 同步最新代码
```

**修改为：**

```
claude/review 先更新 main（git fetch origin + git pull --ff-only origin main），再 rebase main（git rebase main），然后创建 docs/reviews/Pxxx-review.md
```

## 禁止修改项

- 不得修改 `AGENTS.md` 的其他内容
- 不得修改 `PROJECT-SPEC.md`
- 不得修改 `docs/` 下的任何文件
- 不得改动项目结构或其他文件

## 验收标准

1. `AGENTS.md` 中两处模糊的"从 main 同步"已被替换为精确的操作序列
2. 修改后的描述明确等于 PROJECT-SPEC.md 中定义的完整同步流程（更新 main → rebase 工作分支）
3. 修改后的描述不会让 agent 误认为需要执行 `git merge --ff-only origin/main`
4. 其他内容不变

## 测试要求

执行后验证：

```bash
git diff --cached
```

确认仅修改了 `AGENTS.md` 的两处指定行，无其他变更。

## 风险说明

- 注意不要引入其他意外改动
