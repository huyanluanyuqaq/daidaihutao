# P012: 网络资源配置优化（直连下载 + CDN 加速）

## 任务目标

调整所有配置文件的网络资源加载策略：
1. 测速/健康检查地址回退到 `gstatic.com`（P011 改的 cloudflare 撤销）
2. 规则集源恢复 GitHub 地址（P011 改的 jsDelivr 撤销）
3. proxy-providers 和 rule-providers 增加 `proxy: China` 指定直连下载
4. 策略组图标使用 jsDelivr CDN 加速
5. OpenClash 覆写文件使用 jsDelivr 加速

## 修改范围

| 文件 | 操作 |
|---|---|
| `mihomo/config/ProMax.yaml` | 锚点 URL 回退 + 图标 CDN + 添加 `proxy: China` |
| `mihomo/config/Pro.yaml` | 同上 |
| `mihomo/config/Standard.yaml` | 同上 |
| `mihomo/openclash/ProMax_overwrite.conf` | DOWNLOAD_FILE URL → jsDelivr |
| `mihomo/openclash/Pro_overwrite.conf` | DOWNLOAD_FILE URL → jsDelivr |
| `mihomo/openclash/Standard_overwrite.conf` | DOWNLOAD_FILE URL → jsDelivr |

## 禁止修改项

- 不修改 `docs/`、`PROJECT_LESSONS.md`、`PROJECT-SPEC.md`、`AGENTS.md`、`CLAUDE.md`
- 不修改策略组定义、规则顺序、DNS 配置
- 不修改锚点别名（`*oversea`, `*direct` 等）

## 变更 1：测速/健康检查 URL 回退

**位置**：三个 YAML 文件的锚点（`x-base-provider`、`x-url-test`、`x-fallback`、`x-load-balance`）

| 当前值（P011 改的） | 目标值 |
|---|---|
| `https://cp.cloudflare.com/generate_204` | `https://www.gstatic.com/generate_204` |

**逐文件检查**：每个文件中的 `cp.cloudflare.com` 全部替换为 `www.gstatic.com`

---

## 变更 2：规则集源 URL 回退

**位置**：三个 YAML 文件的 `rule-providers` 段

| 当前值（P011 改的） | 目标值 |
|---|---|
| `cdn.jsdelivr.net/gh/huyanluanyuqaq/hutaorules@rules/...` | `raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules/...` |

**注意**：`@rules` 替换回 `refs/heads/rules`，其余路径不变。

---

## 变更 3：proxy-providers 添加 proxy: China

**位置**：三个 YAML 文件的 `proxy-providers` 段

每行增加 `proxy: China` 字段。在 ProMax.yaml 中 3 个 provider，Pro.yaml 和 Standard.yaml 中各 2 个。

**示例**：
```yaml
# 当前
Primary: {<<: *base-provider, url: 'YOUR_PRIMARY_URL', override: {additional-prefix: '[P] '}}
# 目标
Primary: {<<: *base-provider, url: 'YOUR_PRIMARY_URL', override: {additional-prefix: '[P] '}, proxy: China}
```

---

## 变更 4：rule-providers 添加 proxy: China

**位置**：三个 YAML 文件的 `rule-providers` 段

每个 rule-provider 行增加 `proxy: China` 字段。

**示例**：
```yaml
# 当前
Private:  { <<: *rule-set-domain, url: https://raw.githubusercontent.com/.../Private.mrs }
# 目标
Private:  { <<: *rule-set-domain, url: https://raw.githubusercontent.com/.../Private.mrs, proxy: China }
```

需要逐个修改的三文件统计：
- **ProMax.yaml**: 约 25 个 domain + 13 个 ipcidr ≈ 38 处
- **Pro.yaml**: 约 14 个 domain + 9 个 ip = 23 处
- **Standard.yaml**: 约 3 个 domain + 3 个 ip = 6 处

---

## 变更 5：策略组图标使用 jsDelivr

**位置**：三个 YAML 文件的 `proxy-groups` 段的 `icon` 字段

| 当前值 | 目标值 |
|---|---|
| `https://github.com/Koolson/Qure/raw/master/IconSet/Color/*.png` | `https://cdn.jsdelivr.net/gh/Koolson/Qure@master/IconSet/Color/*.png` |

**替换规则**：`github.com/Koolson/Qure/raw/master/` → `cdn.jsdelivr.net/gh/Koolson/Qure@master/`

**注意**：jsDelivr 使用 `@master` 语法（不是 `/refs/heads/master/`）

ProMax.yaml 约 28 个图标，Pro.yaml 约 16 个，Standard.yaml 约 3 个。

## 验收标准

1. 三个 YAML 文件中 `cp.cloudflare.com` 全部替换为 `www.gstatic.com`
2. 三个 YAML 文件中 `cdn.jsdelivr.net/gh/huyanluanyuqaq/hutaorules` 全部回退为 `raw.githubusercontent.com/huyanluanyuqaq/hutaorules/refs/heads/rules`
3. 三个 YAML 文件中每个 `proxy-providers` 条目都有 `proxy: China`
4. 三个 YAML 文件中每个 `rule-providers` 条目都有 `proxy: China`
5. 三个 YAML 文件中图标 URL 全部使用 `cdn.jsdelivr.net/gh/Koolson/Qure@master/...`
6. 三个 `.conf` 文件中 `DOWNLOAD_FILE` URL 全部使用 `cdn.jsdelivr.net`
7. 替换后 YAML 语法验证通过
8. 不要有遗漏——每个文件逐行检视确认

## 风险说明

- 由于 `proxy: China` 指定直连下载，如果规则文件本身需要代理才能访问 GitHub，首次启动时可能拉取失败。但用户明确要求此方案。
- jsDelivr CDN 图标有缓存延迟，新图标最长 24 小时生效。
- P011 的部分变更被撤销，属于正常迭代。
