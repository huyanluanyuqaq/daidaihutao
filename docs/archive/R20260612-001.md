## 3. 修复规范

### 3.1 修复原则

1. **以配置文件为准** — `Pro.yaml` 的 `rule-providers` 声明为唯一权威来源
2. **不修改配置** — 只对齐 `rules/` 目录，不改动 `mihomo/config/cn/` 中的任何文件
3. **不碰其他目录** — `rules/singbox/`、`rules/surge/` 保持现有不变
4. **外部推送模式** — 修复后的 `rules/` 目录由 `P003_rules-builder` 维护

### 3.2 修复步骤

#### Step 1: 删除多余文件

- `rules/mihomo/domain/` — 删除 20 个未引用 `.mrs`
- `rules/mihomo/ip/` — 删除 3 个未引用 `.mrs`
- `rules/mihomo/*.txt` — 删除 24 个未引用 `.txt`

#### Step 2: 补充缺失文件

从上游源 `https://github.com/666OS/rules`（release 分支）下载 15 个缺失 `.mrs` 文件：
- domain/ 4 个：`AIIP.mrs`、`ChinaIP.mrs`、`EmbyIP.mrs`、`NetflixIP.mrs`
- ip/ 11 个：`PrivateIP.mrs`、`TelegramIP.mrs`、`SocialMediaIP.mrs`、`AIIP.mrs`、`NetflixIP.mrs`、`EmbyIP.mrs`、`StreamingIP.mrs`、`ProxyIP.mrs`、`ChinaIP.mrs`、`GoogleIP.mrs`、`FacebookIP.mrs`

#### Step 3: 修正归属

将 `domain/` 中以 `IP.mrs` 结尾的 4 个文件移到 `ip/`。

#### Step 4: 验证

```bash
echo "domain mrs: $(ls rules/mihomo/domain/*.mrs | wc -l)"  # 预期: 19
echo "ip mrs: $(ls rules/mihomo/ip/*.mrs | wc -l)"          # 预期: 11
```

### 3.3 最终目录结构

```
rules/
├── mihomo/
│   ├── domain/          # 19 个 .mrs 文件
│   ├── ip/              # 11 个 .mrs 文件
│   └── *.txt            # 19 个 .txt 文件（与 domain/ 一一对应）
├── singbox/             # 保持现有不变
└── surge/               # 保持现有不变
```

---

## 4. 注意事项

1. `rules/singbox/` 和 `rules/surge/` 不受此次修复影响
2. 所有规则文件使用 mihomo 标准 `.mrs` 二进制格式
3. 修复完成后配置文件应能通过 `mihomo rule` 验证
4. `P003_rules-builder` 接入后，规则文件由其自动生成并推送，本仓库不再执行规则同步或编译
